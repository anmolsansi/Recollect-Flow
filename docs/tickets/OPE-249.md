# OPE-249 implementation evidence

- Private R2 object keys are opaque and never returned as public bucket URLs.
- Controlled upload validates auth, state, expiry, configured maximum, exact declared/actual size, MIME allowlist, magic signature and SHA-256.
- Finalization rechecks R2 size, MIME and checksum metadata and is idempotent.
- `POST /api/v1/captures` accepts and atomically links one finalized attachment.
- Authorized download streams private bytes; anonymous access fails.
- Admin deletion removes R2 bytes and tombstones D1 state.
- Hourly cleanup removes expired unlinked objects and is idempotent.
- Usage endpoint reports active count/bytes by status.
- Migration copies existing attachment rows before removing the legacy table.
- Live local evidence: PDF init/upload/finalize/link/download passed with byte-exact output; D1 reported `linked` and anonymous download returned `401`.
- Production evidence: private PDF init/upload/finalize/link/download passed with byte-exact SHA-256; D1 reported `linked`, anonymous download returned `401`, authenticated deletion returned `204`, D1 tombstoned the attachment, and the private R2 bucket returned to zero objects.
